Overview

What is NIKA?

NIKA is a unified platform that bridges AI agents with realistic network environments, enabling reproducible evaluation and real-time troubleshooting.

Benchmark Suite

54 curated network incidents covering link failures, host failures, routing misconfigurations, resource contention, and attacks. Combined with 5 scalable topologies, NIKA yields 640 distinct troubleshooting incidents. Incidents can be further extended by randomising failure locations and composing multiple issues.

Orchestration Platform

A modular plug-and-play platform that connects AI agents with the network environment via MCP servers, enabling real-time troubleshooting in realistic conditions and providing a human-facing interface to monitor agent performance.

NIKA system architecture
Features

Built for AI + Networking Research

Kathará-based Environment

Standardised network emulation built on Kathará, enabling reproducible and realistic network topologies with real routing daemons.

MCP Tool Integration

Model Context Protocol servers expose network operations — ping, iperf, BGP/OSPF config, flow rules, telemetry — as callable tools for any MCP-compatible agent.

Pre-built Scenarios & Faults

54 realistic faults spanning 6 categories, ready to inject into any scenario at any topology scale with a single CLI command.

Reproducible Evaluation

Automatic evaluation via numeric metrics, LLM-as-judge scoring, and CSV publishing, ensuring fair and consistent comparisons across agents and models.

Multiple Topology Sizes

Scalable scenarios support small, medium, and large topology tiers — campus networks, data centres, ISP backbones, SDN fabrics, and P4 testbeds.

Extensible by Design

Plug in custom MCP servers, define new scenarios via Kathará's Python API, add custom agents, or hook into LangSmith / Langfuse / Laminar for observability.

NIKA CLI

A unified command-line interface to manage environments, inject faults, run agents, evaluate results, and drive traffic — all from a single tool.

Public Dataset

Over 900 reasoning traces from multiple AI agents across the full benchmark, publicly available on Zenodo for analysis and replication.

Quick Start

Up and Running in Minutes

Requires Kathará and Python ≥ 3.12. Uses uv for dependency management.

1

Install

Clone the repository and install dependencies.

bash 
# Clone the repository
git clone https://github.com/sands-lab/nika
cd nika

# Install dependencies with uv (recommended)
uv sync

# Or with pip
pip install -e .
2

Configure

Create a .env file with your API keys and observability settings.

.env 
# LLM API keys
OPENAI_API_KEY=<your-key>
DEEPSEEK_API_KEY=<your-key>

# Optional: observability (LangSmith)
LANGSMITH_TRACING=true
LANGSMITH_API_KEY=<your-key>
LANGSMITH_PROJECT=<project-name>
3

Run a Scenario

List available scenarios and start the network environment.

bash 
# List available scenarios
nika env list

# Start a scalable scenario (s / m / l)
nika env run dc_clos_bgp -t s

# Inject a fault
nika failure inject bgp_asn_misconfig

# Run an AI agent
nika agent run -a react -b openai -m gpt-4o-mini -n 20
4

Evaluate

Measure performance with numeric metrics and an LLM judge, then publish results.

bash 
# Compute numeric metrics
nika eval metrics

# Score with an LLM judge
nika eval judge -b openai -m gpt-4o-mini

# Publish summary CSV and teardown
nika eval publish
MCP Servers & Tools

Network Operations as Agent Tools

NIKA exposes network operations through MCP servers, letting any MCP-compatible agent (Claude, GPT-4, Gemini, open-source LLMs…) directly interact with the network environment.

Base Kathará Server

  • get_reachability — Ping all host pairs
  • iperf_test — Bandwidth test between two hosts
  • systemctl_ops — Start / stop / restart services
  • get_host_net_config — Retrieve host network config
  • nft_list_ruleset — Dump nftables ruleset

BMv2 P4 Server

  • bmv2_get_log — Retrieve BMv2 switch logs
  • bmv2_get_counter_arrays — Read counter arrays

FRRouting Server

  • frr_get_bgp_conf — Retrieve BGP configuration
  • frr_get_ospf_conf — Retrieve OSPF configuration

INT / InfluxDB Server

  • influx_list_buckets — List InfluxDB buckets
  • influx_get_measurements — Retrieve measurements
  • influx_query_measurement — Query telemetry data

Generic Server

  • google_search — Web search for context

Task Management Server

  • list_avail_problems — List available problems
  • get_submission_template — Get submission template
  • submit — Submit a solution

More tools coming soon. You can also plug in any custom MCP server — browse mcp.so for the community catalogue.
